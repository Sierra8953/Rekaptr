## 2024-06-25 - Avoid HashSet clone for batch drain operations
**Learning:** When needing to iterate over and consume elements in a GPUI component state collection (like a `HashSet<String>` containing paths) during an event callback, using `.clone()` to copy the entire collection just to iterate over it and then calling `.clear()` on the original is a performance anti-pattern. This causes an unnecessary O(N) heap allocation and copies all string contents.
**Action:** Use `.drain()` directly on the collection instead of `.clone()` + `.clear()`. This completely avoids the allocation and string cloning while achieving the exact same destructive iteration and clearing effect. Ensure the borrow checker is happy by ensuring `.drain()` iteration completes before any subsequent mutating calls on the struct.
