## 2024-05-24 - Avoid Double Lookups and Filter Early in EnumWindows Hot Path
**Learning:** In the Windows `EnumWindows` callback `enumerate_windows_callback`, querying the process tree (`sysinfo::System::process`) and window text (`GetWindowTextW`) are expensive OS-level operations. The original code performed a `sysinfo` process lookup for every visible window, followed by getting the title, and then immediately performed the exact same `sysinfo` process lookup *again* to construct the `WindowInfo` struct. Furthermore, it performed process lookups for windows with no text. `GetWindowTextW` returns an integer length where 0 means either an error or no text, allowing a single check to filter out nameless windows before performing expensive process lookups.
**Action:** When filtering OS resources in hot paths like `EnumWindows`, query and filter based on the cheapest properties first (e.g., window visibility and text length) to avoid invoking expensive APIs (like `GetWindowThreadProcessId` or `sysinfo` process tree traversal) on irrelevant objects. Ensure that expensive data retrieval is done only once and the result is reused instead of querying it again.
