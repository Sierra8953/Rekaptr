## Design / UX

### [ ] Redesign the Clips page
The clips library layout needs a visual pass.

- Surfaces: `src/ui/clips.rs` — `render_clips`, `render_clips_hero`,
  `render_clips_body`, `render_clips_group_row`, `render_clip_card_advanced`,
  `render_clips_filter_rail`, `render_clips_top_bar`.
- Goals (to refine):
  - Tighten the hero + grouped-rows hierarchy so the page reads as one library,
    not stacked sections.
  - Revisit card sizing/spacing, thumbnail aspect, hover and selection states.
  - Make the filter rail (All / Favorites / Recent / per-game) feel consistent
    with the rest of the app's chrome.
  - Empty / loading states.
- Note: the per-frame render cost here was just reduced (grouping + path
  allocations), so layout changes shouldn't reintroduce per-clip allocations or
  quadratic scans — keep using `Clip.path_str` and the single-pass grouping.

### [ ] Redesign the clip mini player
The in-page clip preview/player needs a design pass.

- Surface: `src/ui/clips.rs` — `render_mini_player` (and `clip_to_preview` flow).
- Goals (to refine):
  - Cleaner transport controls, scrubber, and close affordance.
  - Better framing/sizing relative to the clips grid.
  - Consistent styling with the main dashboard player.

## Artwork

### [x] Stop fetching artwork that's never shown
Steam artwork is now limited to the variants we actually display:

- [x] Removed the **portrait** variant entirely (it was fetched into
  `portrait_cache` but never rendered). Dropped the cache, `find_steam_artwork_portrait`,
  `fetch_portrait_artwork`, and the portrait download in `prefetch_artwork`.
- Remaining Steam artwork:
  - Source / source cards → **blurred library hero** (`find_steam_artwork` +
    `ensure_blurred`, `src/ui/dashboard.rs`).
  - Dashboard video overlay → transparent **logo** (`find_steam_logo`) — kept.

### [ ] Refine source-card artwork (visual)
Now that fetching is trimmed, the source-card art still needs a look-and-feel pass:

- Fallback when there's no Steam match (currently falls back to an `avatar_tint`).
- Blur quality/consistency via `ensure_blurred`, plus sizing/crop of the hero.
